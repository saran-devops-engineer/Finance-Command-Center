# Design Principles

The visual direction is premium, calm, minimal, mobile-first, and Apple-inspired.

Do not redesign the product from scratch. Improve usability while preserving the existing visual language.

## Visual Language

Use:

- warm paper-like backgrounds
- generous whitespace
- rounded cards
- soft shadows
- serif display headings
- readable sans-serif body text
- dark pill actions
- subtle dividers
- restrained color
- calm financial language

Avoid:

- dense dashboards
- table-heavy layouts
- harsh red/green overload
- noisy charts
- crowded widgets
- generic SaaS dashboard styling

## Mobile First

Design for phone screens first.

Requirements:

- comfortable thumb targets
- bottom navigation
- safe-area compatibility
- smooth vertical scrolling
- responsive tablet and desktop behavior
- PWA installability

## Information Hierarchy

Each screen should lead with the most important decision.

Home should prioritize:

1. financial status
2. available money
3. today's best decision
4. urgent dues
5. active portfolio summary

## Tone

The app should feel like a calm financial companion.

Use:

- "Review today"
- "Keep money ready"
- "Estimated interest saved"
- "Available after commitments"

Avoid:

- "Bad score"
- "Failure"
- "You overspent again"
- shame-based language

## Loan UI

Loans should feel like premium financial cards.

Never use raw data tables for loan overview experiences.

Loan cards should show:

- name
- lender
- outstanding balance
- progress
- interest rate
- next due date
- clear action path
