# UX Review

## Current Strengths

- Clear mobile-first direction.
- Calm, premium visual language.
- Home focuses on financial status and action.
- Loans use card-based presentation instead of tables.
- Onboarding prevents an empty first dashboard.
- Local-first privacy is visible in Profile.

## Current Risks

- Financial projections are estimates and need clear explanation.
- Onboarding can still become long if more modules are added.
- Users need stronger edit/delete flows.
- Offline install behavior needs real device testing.
- Empty states need continued polish.
- Some date display is still raw ISO format.

## Home Screen Review

Home should answer:

> Am I financially okay today?

It should prioritize:

- available money after commitments
- contextual financial status
- today's best decision
- urgent dues
- active portfolio summary

Avoid:

- showing every loan
- generic status copy
- dashboard clutter
- decorative charts without action

## Loans Review

Loans should answer:

> Which loan needs attention?

Loan experiences should emphasize:

- highest interest burden
- upcoming EMI
- payoff progress
- prepayment impact
- recent payments

## Money Review

Money should answer:

> How much money can I safely use?

It should explain available money in plain language.

## Insights Review

Insights should answer:

> What should I do next?

Insights should be specific, explainable, and conservative.

## Profile Review

Profile should answer:

> How is my data managed?

It should make local-first privacy clear and prepare users for future backup/restore.

## UX Backlog

- Add Recent Activity to Home.
- Add edit/delete loan flows.
- Add friendly date formatting.
- Add stronger onboarding validation.
- Add empty state actions.
- Add PWA install education.
- Add export/import in V1.1.
