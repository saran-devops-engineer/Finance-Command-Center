# Finance Command Center

# Document 010 — Domain Model

**Version:** 1.0

**Status:** Approved

**Priority:** Highest

---

# Purpose

This document defines the business language used throughout Finance Command Center.

Every screen, feature, calculation, recommendation, notification and future AI capability must use these definitions.

These definitions are the single source of truth.

---

# Domain Philosophy

Finance Command Center is built around financial decisions.

Everything in the application exists because it helps users understand or improve one financial decision.

---

# Core Domains

Finance Command Center consists of six core domains.

1. Cash Flow

2. Debt

3. Financial Health

4. Decisions

5. Planning

6. Data

Every future feature belongs to one of these domains.

---

# Entity: Loan

Definition

A financial obligation where money is borrowed and must be repaid according to agreed terms.

Examples

Home Loan

Gold Loan

Personal Loan

Vehicle Loan

Education Loan

Hand Loan

Credit Card EMI

Consumer EMI

---

A Loan contains

Loan Name

Loan Type

Lender

Principal

Outstanding Balance

Interest Rate

Interest Type

EMI

Tenure

Remaining Tenure

Start Date

End Date

Status

Payment History

Documents

Notes

---

# Entity: EMI

Definition

The scheduled recurring payment made toward a loan.

EMI consists of

Interest Component

Principal Component

Every EMI changes the Outstanding Balance.

---

# Entity: Outstanding Balance

Definition

The remaining principal that has not yet been repaid.

Outstanding Balance is never

Original Loan

minus

EMIs

using shortcuts.

It is always derived from the amortization schedule.

---

# Entity: Interest

Definition

The cost paid for borrowing money.

Interest may be

Reducing Balance

Flat

Bullet

Simple

Compound

depending on the loan type.

---

# Entity: Monthly Commitment

Definition

Any payment that the user must make regardless of personal choice.

Examples

EMI

Rent

Insurance

School Fees

Subscriptions

Electricity

Internet

Phone

These commitments are used when calculating Available Money.

---

# Entity: Income

Definition

Money received by the user.

Types

Salary

Business

Rental

Freelance

Interest

Passive Income

Future income types should extend this entity.

---

# Entity: Expense

Definition

Money voluntarily spent by the user.

Examples

Food

Travel

Shopping

Entertainment

Medical

Dining

Fuel

Unlike Commitments,

Expenses are partly controllable.

---

# Entity: Available Money

Definition

The amount of money that remains after mandatory commitments and the configured emergency buffer have been accounted for.

Available Money is not the same as Bank Balance.

Available Money answers one question:

"How much money can I safely allocate without risking upcoming obligations?"

---

# Entity: Emergency Buffer

Definition

Money intentionally left untouched to handle unexpected expenses.

The buffer is deducted before Available Money is calculated.

---

# Entity: Recommendation

Definition

A recommended financial action generated by the Recommendation Engine.

Every Recommendation must include

Reason

Expected Benefit

Trade-offs (if any)

Confidence

Recommended Action

Recommendations must always be explainable.

---

# Entity: Insight

Definition

An observation.

Insights explain.

Recommendations advise.

Examples

Insight

Gold Loan has the highest monthly interest.

Recommendation

Consider making a ₹50,000 prepayment.

---

# Entity: Simulation

Definition

A temporary financial scenario.

Simulations never modify actual data.

A Simulation compares

Current State

↓

Hypothetical Change

↓

Future Projection

↓

Difference

↓

Recommendation

---

# Entity: Financial Health

Definition

A high-level assessment of the user's current financial position.

Financial Health is calculated using multiple factors rather than a single metric.

Future versions may include

Debt Burden

Cash Flow

Emergency Buffer

Savings Rate

Commitment Ratio

Net Worth Trend

---

# Entity: Net Worth

Definition

Assets

minus

Liabilities.

Investments will contribute in future versions.

---

# Entity: Asset

Definition

Anything owned that has financial value.

Future examples

Cash

Property

Gold

Mutual Funds

Stocks

Provident Fund

Vehicles

---

# Entity: Liability

Definition

Any financial obligation reducing Net Worth.

Loans are liabilities.

---

# Final Rule

If a future feature introduces a new financial concept,

it must first be added to the Domain Model before implementation.

The Domain Model is the business vocabulary of Finance Command Center.
